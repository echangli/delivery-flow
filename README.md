# delivery — плагин Claude Code для флоу доставки

Пошаговый флоу, который ведёт человека (даже без опыта в коде) от идеи или бага до готового
**MR на ревью**: дизайн → код → проверка → QA → Merge Request. Работает поверх **Claude Code**.

**Автор:** Eduard Changli ([@echangli](https://github.com/echangli)).

---

## 🚀 Как начать

### Проще всего — пусть Claude сам проведёт

1. Установи **Claude Code** (десктоп-приложение: https://claude.com/download → вкладка **Code**) или открой терминал `claude`.
2. Открой любой чат и **вставь туда ссылку на этот репозиторий:**
   ```
   https://github.com/echangli/delivery-flow
   ```
   и напиши: **«поставь этот плагин для Claude Code и проведи меня по настройке».**
3. Дальше Claude всё объяснит по шагам: подскажет команды установки, задаст пару вопросов про твой
   проект и поможет настроить. Тебе останется соглашаться и отвечать.

> Не хочешь даже ссылку искать — просто **перетащи файл `README.md` в чат** и напиши то же самое.

### Или вручную — три команды

В терминале `claude` (репозиторий публичный — **аккаунт GitHub и VPN не нужны**):
```
/plugin marketplace add https://github.com/echangli/delivery-flow.git
/plugin install delivery@delivery-toolkit
/plugin install superpowers@claude-plugins-official
```
При установке выбери scope **User** (плагин будет доступен во всех проектах). `superpowers` —
обязательная зависимость (публичный официальный плагин).

## Что это

- **Флоу доставки** с тремя треками: **A** — новая фича (с дизайном), **B** — багфикс, **C** — вход
  в готовый MR / правка на стейдже / тексты. Финал — Merge Request, готовый к ревью.
- **QA-скиллы**: разбор чужого MR, смоук своего MR на стейдже, durable-автотесты (Playwright).
- Запуск — команда `/deliver` или просто опиши задачу в чате.

Подходит всем — продакту, QA, дизайнеру, аналитику: не нужно быть кодером.

## Настройка под свой проект (один раз)

Это **шаблон**: механика флоу готова, а места, специфичные для проекта (репозиторий, команды сборки,
дизайн-система, продукт), помечены `⟪ADAPT: …⟫`. Перед первым использованием их нужно заполнить —
**это делает один техничный человек в команде, ~1–2 часа.**

Проще всего: открой папку плагина в Claude Code и скажи **«прочитай `ADAPT.md` и помоги пройти по меткам
⟪ADAPT:⟫ под наш проект»** — Claude проведёт. Полный гайд — [`ADAPT.md`](ADAPT.md); подробный онбординг и
использование — [`plugins/delivery/README.md`](plugins/delivery/README.md).

## Обновления

Версия привязана к коммиту репозитория — обновление = подтянуть свежий `main`. Репозиторий публичный,
поэтому апдейты приходят без токенов и логина.

> ⚙️ **Куда вводить команды ниже.** `/plugin` и `/reload-plugins` — это команды **самого Claude Code**,
> а не обычного терминала (это не shell-команды). Открой Terminal, набери `claude` и нажми Enter —
> запустится Claude Code; команды со слэшем печатай **прямо в его строке ввода**.

**Включить автообновление — один раз, по шагам:**
1. В Terminal запусти `claude` (откроется Claude Code).
2. Набери `/plugin` и нажми Enter — откроется меню плагинов.
3. Стрелками перейди в раздел **Marketplaces** и выбери **`delivery-toolkit`** (Enter).
4. Найди пункт **Enable auto-update** и включи его (Enter).

Готово. Теперь при каждом запуске Claude Code свежая версия подтягивается сама. Когда появится
уведомление об обновлении — набери **`/reload-plugins`** (или просто перезапусти Claude Code), и новая
версия активируется.

**Обновить вручную** (если автообновление не включал) — в том же Claude Code набери две команды:
```
/plugin marketplace update delivery-toolkit
/reload-plugins
```
