# delivery — плагин Claude Code для флоу доставки

Пошаговый флоу, который ведёт человека (в т.ч. не-кодера) от идеи или бага до **готового draft-MR,
одобренного фронтендером**: дизайн → код → локальная проверка → QA-смоук → MR. Оформлен как marketplace
Claude Code с одним плагином `delivery`.

**Автор:** Eduard Changli ([@echangli](https://github.com/echangli)).

> 🔒 **Приватный репозиторий.** Доступ выдаётся по запросу. При использовании и форках, пожалуйста,
> сохраняйте указание авторства.

---

## Что внутри

- **`delivery`** — плагин с оркестратором (треки **A** — фича, **B** — багфикс, **C** — вход в середину),
  QA-скиллами (`qa_mr`, `delivery_qa_smoke`), durable-автотестами (`autotest`/`autotest_run`/`selectors_sync`),
  прототипированием (`delivery_prototype`) и хуками.
- Это **обобщённый скелет**: рабочая механика на месте, а всё, что специфично для конкретного проекта,
  вынесено в помеченные плейсхолдеры `⟪ADAPT: …⟫` — их вписывают под себя.

## С чего начать

| Хочешь… | Открой |
|---|---|
| **Адаптировать под свой проект** (один раз, ~1–2 часа) | [`ADAPT.md`](ADAPT.md) — гайд форка: что вписать, реестр всех мест, как поставить |
| **Поставить и пользоваться** (после адаптации) | [`plugins/delivery/README.md`](plugins/delivery/README.md) — онбординг, установка, треки |

Быстрее всего — открыть папку в Claude Code и сказать: «прочитай `ADAPT.md` и помоги пройти по меткам
`⟪ADAPT:⟫` под наш проект».

## Быстрая установка (после того как адаптируешь и запушишь свой форк)

```
/plugin marketplace add https://github.com/echangli/delivery-flow.git
/plugin install delivery@delivery-toolkit
/plugin install superpowers@claude-plugins-official   # обязательная зависимость
```
