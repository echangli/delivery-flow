# Delivery — быстрый старт (для чата)

Плагин, который доведёт твою фичу/баг до готового MR (ревью дизайнера/QA/фронтендера).

**Где запускать:** сам флоу — в **десктоп-приложении Claude Code** (вкладка **Code**), не в вебе и не «навсегда в терминале». Нет приложения → поставь с **https://claude.com/download** → войди → вкладка **Code** → **Local** → папка проекта. Терминальный `claude` нужен **только** для установки плагина (шаг 1).

**🤖 Проще всего:** открой Claude Code, **прикрепи ему README плагина** ([README.md](https://github.com/echangli/delivery-flow/blob/main/plugins/delivery/README.md)) и напиши «проведи меня по этому файлу — помоги поставить и настроить доступы». Дальше он сам по шагам. Либо вручную:

Шаги:

1. **Установка плагина (один раз, в терминальном `claude`)** — репо **публичный**, ставится тремя командами, аккаунт GitHub не нужен:
   - Запусти `claude` в терминале и внутри:
     `/plugin marketplace add https://github.com/echangli/delivery-flow.git`
     `/plugin install delivery@delivery-toolkit` → на вопрос про scope выбери **`User`** (во всех проектах).
   - Поставь обязательную зависимость: `/plugin install superpowers@claude-plugins-official` (тоже scope `User`).
   - Включи автообновление один раз: `/plugin` → `Marketplaces` → `delivery-toolkit` → `Enable auto-update`. (Иначе вручную: `/plugin marketplace update delivery-toolkit` → `/reload-plugins`.)
   - **Отдельно, когда дойдёшь до кода:** токен git-хоста **твоей команды** (`⟦GIT_HOST⟧`) для push ветки и авто-деплоя стейджа — см. §3.5 README. Это не то же самое, что скачать плагин (плагин публичный, доступ не нужен).
2. Дальше — **в десктопе**: запусти `/deliver` (или просто опиши фичу/баг — подхватит сам).
3. Опиши подробно + приложи скрины/макеты/ссылки на карточку (Notion).
4. Пройди шаги: дизайн+прототип (для фичи — **несколько вариантов на выбор дизайнеру**) → реализация → локальная проверка → QA-смоук.
5. Получи **draft-MR с уже заполненным описанием** → отдай на ревью фронтендеру.
6. После апрува **мержат и деплоят QA или фронтендер** (не ты).

Нет доступа (дев-VPN, аккаунт на git-хосте команды, фронт-репо ≥ Developer, MCP)? Напиши в `⟦CHAT_CHANNEL⟧` — флоу подскажет, что именно просить. (Сам плагин публичный — для него доступ не нужен.)
