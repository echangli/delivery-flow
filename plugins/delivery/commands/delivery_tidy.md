Наведи порядок в рабочей папке флоу доставки по канону
`delivery_orchestrator/references/workspace-layout.md` (§ 4 «Навести порядок»).

Кратко:

1. **Определи рабочую папку флоу.** Если задачи ведутся «где попало» (Desktop, Downloads, корень home) —
   предложи завести **отдельную папку под флоу** (напр. `~/<команда>-delivery/`) и дальше вести задачи там.
2. **Найди свалку:** плоские `*.html`-прототипы, `delivery-notes-*.md`, папку `prototypes/`, скрины,
   спеки/планы **без группировки по задачам**.
3. **Сгруппируй по задачам** (по слагу/дате в именах) в `tasks/<тикет>-<слаг>/` (без тикета — `<слаг>`)
   с короткими именами: `design.md` / `plan.md` / `prototype.html` / `screenshots/` / `notes.md` /
   `retro.md` / `locales/`.
4. **Покажи план переноса (что → куда) и дождись подтверждения** — перемещать только после «ок».
5. **Перенеси:** под git — `git mv`, иначе `mkdir -p tasks/<слаг> && mv …`. **Ничего не удаляй** — только
   перемещай. Неоднозначное (непонятно, к какой задаче) — оставь на месте и спроси.
6. Если в `CLAUDE.md`/`AGENTS.md` воркспейса задана **своя** раскладка — приводи к ней, а не к дефолту.
7. **Итог:** короткий отчёт «что куда собрано»; напомни начинать новые задачи сразу в `tasks/<слаг>/`.

**Guardrails:** не трогай git-репозитории проектов (код-репо / worktree) — уборка касается только
артефактов флоу в рабочей папке; секреты (`.env`) не коммить и не свети в чат.
