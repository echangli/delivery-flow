# Конвенции autotest_impact — сигналы, греп-рецепты, уверенность, шаблон отчёта

Детали для скилла `autotest_impact`. Карта устройства сьюты (слои `spec → steps → selectors → fixtures`,
формат id `layer:module:component`, правило generic-элементов ДС «контейнер + роль») —
`../../autotest/references/autotest-conventions.md` (**не дублируем**). Здесь — как из диффа получить
сигналы, чем грепать сьюту, как градуировать уверенность и как оформить отчёт.

## Сигналы S1–S5 и греп-рецепты

Дифф: `git diff origin/main...HEAD -- ⟦APP_PATH⟧`
(+ shared-пакеты с `generateTestId`). Сьюту грепаем в `<frontend>/⟦E2E_PATH⟧/src`.

| # | Сигнал | Что берём из диффа (удалённые `-` строки) | Греп по сьюте |
|---|---|---|---|
| S1 | Убран/переименован `data-test` | `-` строки с `data-test="…"`, `useTestId('…')`, `generateTestId(…)` → набор потерянных/изменённых id | точный литерал id: `grep -rn "<id>" <e2e>/src` (обычно `selectors.ts`) |
| S2 | Компонент с тест-id переехал/удалён/переименован | удалённые/переименованные файлы и export'ы, содержавшие тест-id | `grep -rn "<модуль/группа>" <e2e>/src` — группы `*TestIds` и шаги |
| S3 | Изменён/удалён англ. i18n-лейбл/строка | дифф английского i18n-исходника (default-сообщение в коде / файл локали `en_US`) | `grep -rn "<текст>" <e2e>/src` — `getByText('<текст>')`, `{ name: '<текст>' }` |
| S4 | Изменён флоу/состояние | удаление/замена интерактивного узла (оверлей/шаг/порядок) рядом с покрытой поверхностью | шаги `steps.ts` этой поверхности — последовательность вызовов |
| S5 | Изменён роут/deep-link | дифф констант роутов, deep-link-параметров | `page.goto(...)`, navigate-шаги, deep-link в спеках |

**Рецепт-пример (S1):**
```bash
# потерянные id из диффа (формат layer:module:component)
git diff origin/main...HEAD -- <app> | grep -E '^-' | grep -oE "[a-z]+:[a-zA-Z]+:[a-zA-Z-]+"
# каждый id → где в сьюте
grep -rn "features:orderForm:reset-btn" <frontend>/<e2e>/src
```

## Границы статики (почему нужен прогон)
- **`generateTestId` из констант:** id может собираться композицией (`generateTestId(layer, module, name)`);
  переименование константы меняет эмитируемый id **без** литерального диффа id-строки → S1 не поймает
  точно. Смягчение: грепнуть аргументы `generateTestId(...)` в диффе + прогон-подтверждение.
- **Не вся копия ассертится:** тесты предпочитают `data-test` тексту; S3 = кандидат, не приговор.
- Поэтому 🟡/🟢 — «нужен глаз/прогон», а 🔴 держим только за детерминированный слом id.

## Рубрика уверенности
- 🔴 **High** — id-литерал из `selectors.ts` убран/переименован в продукте, product-side алиаса нет →
  спека упадёт. *Пример:* `resetBtn: 'features:orderForm:reset-btn'` в `selectors.ts`, а во фронте id стал
  `…:reset-default-btn`. (Также: 🟡, подтверждённый прогоном как red.)
- 🟡 **Medium** — generic «контейнер + роль» (лейбл/скоуп), i18n-текст (S3), либо элемент уехал за новое
  состояние/флаг. *Пример:* шаг ищет `getByRole('button', { name: 'Reset to defaults' })`, а лейбл стал
  `'Reset settings'`.
- 🟢 **Low / heads-up** — флоу-дрейф (S4) рядом с покрытой поверхностью, прямого слома локатора нет.
  *Пример:* убрали промежуточный confirm-оверлей — шаг `confirmReset` может стать лишним.

## Шаблон отчёта (`autotest-impact.md`)
```markdown
# Автотест-импакт — <ветка/MR>

**Вердикт:** N спек под риском (X 🔴 / Y 🟡 / Z 🟢)   [или: Импакт на автотесты не обнаружен]
**Дифф:** <краткое что изменилось>   **Прогон:** <был на слоте / [не подтверждено: нет слота]>

## Находки
### 🔴 <спека> — <короткая причина>
- **Файл:** `<e2e>/src/<surface>/<name>.spec.ts` (селектор: `selectors.ts:<line>` `<group>.<key>`)
- **Почему:** <корневая причина сигналом S#>
- **Рекомендация QA:** <конкретная правка selectors.ts/шага>
- **Дешевле (product-side):** <сохранить id/лейбл → правка теста не нужна>   [если применимо]
- **Прогон:** <red подтверждён / green — риск снят / не прогонялось>

### 🟡 … (аналогично)
```
При массовом диффе — **группируй находки по поверхности**, не плоской простынёй.

Ниже отчёта — **сопроводиловка для QA** (1–3 предложения: что за изменение, сколько спек, суть рисков),
готовая к копированию в чат-канал. Агент сам никуда не постит.

## Прогон-подтверждение (делегируем `autotest_run`)
- Гейт: слот отвечает (`SLOT_URL` из `.env`/авто-деплой — как в `autotest_run`/`delivery_qa_smoke`), devVPN/доступ.
- Точечно: кандидатные спеки, `--project=desktop` первым (tablet/phone при нужде). **Только прогон** —
  починку `autotest_run` не запускаем.
- Сверни: red → подтверждает/повышает (🟡→🔴); green по 🟡 → сними как ложную с пометкой «прогон green».
- Нет слота/браузеров → `[не подтверждено прогоном: нет слота]`, отчёт остаётся на статике.
- Реальный ассерт-фейл (не локатор) → возможно, баг фронта: не подгонять, зафиксировать (Трек B).

## Edge-кейсы
- **Дифф трогает только тесты** (сьюта, продукт не менялся) → «нет изменений продукта, импакт-чек не нужен».
- **Новый `data-test` добавлен** (не убран) → не риск для существующих спек; в отчёт не идёт (это `selectors_sync`/`autotest`).
- **Generic-элемент ДС** (`Button`/`Input` без уникального id) → тест локейтит контейнером+ролью: правка
  лейбла ломает name, контейнера — scope; обе причины в находку.
- **Переиспользуемый шаг чужой поверхности** → трейс до всех спек, зовущих шаг (не только «родной» поверхности).
- **Массовый рефактор** → группировать находки по поверхности.
- **Нет доступа к репо/диффу** → сообщить, не «висни», не выдумывать матчи.
