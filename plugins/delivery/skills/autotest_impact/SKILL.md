# /autotest_impact — задел ли дифф автотесты e2e-сьюты

Статически сверяет дифф фронта (`⟪ADAPT: путь к приложению фронта, напр. apps/web-client⟫`) с durable-сьютой
`<frontend>/⟪ADAPT: путь к e2e-сьюте, напр. e2e/web⟫` и выдаёт **отчёт для QA**: какие спеки под риском,
почему, с какой уверенностью, что править. **Сам тесты не правит** — в этом суть: правки идут QA
хэндоффом, а не подгонкой чужих тестов под своё изменение.

Карта устройства сьюты (слои `spec → steps → selectors → fixtures`, формат id, правило generic-элементов
ДС) — `../autotest/references/autotest-conventions.md`. Сигналы, греп-рецепты, рубрика уверенности и
шаблон отчёта — `references/impact-conventions.md`. **Прочитай оба перед разбором** — не выводи устройство
из головы.

## Вход
Ветка/дифф фронта — своя (в рамках доставки) или чужая (ревью входящего MR). Продуктовый контекст
поверхностей — primer `${CLAUDE_PLUGIN_ROOT}/context/platform.md`; специфика ключевой под-системы —
`${CLAUDE_PLUGIN_ROOT}/context/⟪ADAPT: имя файла контекста ключевой фичи⟫.md`.

## Гейт
- Есть доступ к фронт-репо и диффу против `origin/main` (worktree/ветка). Нет доступа/диффа → сообщи,
  **не «висни»** и не выдумывай матчи.
- Целевой репо — **основной фронт** с e2e-сьютой. **Опциональный второй репо/лендинг** (иной стек, напр.
  Astro) e2e-сьюты обычно не имеет → выведи «N/A для этого репо» и заверши.
- Дифф не трогает `⟪ADAPT: путь к приложению фронта, напр. apps/web-client⟫` (только тесты/доки/конфиг) →
  «нет изменений продукта, импакт-чек не нужен».

## Процесс
1. **Дифф.** `git diff origin/main...HEAD -- ⟪ADAPT: путь к приложению фронта, напр. apps/web-client⟫`
   (+ shared-пакеты, эмитящие тест-id, напр. хелпер `generateTestId`).
2. **Сигналы.** Извлеки S1–S5 (детали и рецепты — `references/impact-conventions.md`): убранный/
   переименованный `data-test`/`useTestId`/`generateTestId` (S1); переехавший/удалённый компонент с
   тест-id (S2); изменённый английский i18n-лейбл/строка (S3); изменённый флоу/состояние (S4);
   изменённый роут/deep-link (S5).
3. **Греп сьюты.** По каждому сигналу пройди `⟪ADAPT: e2e-сьюта⟫/src` рецептом из конвенций (точный
   id-литерал в `selectors.ts`; текст-литерал в `steps.ts`/спеках; и т.д.). Только фактические матчи — не
   выдумывай.
4. **Трейс до спеки.** `selectors.ts` группа → её вызов в `steps.ts` → `*.spec.ts`. Учитывай
   переиспользование шагов чужими поверхностями (шаги одной поверхности внутри спеки другой) — дойди до
   **всех** зависящих спек.
5. **Уверенность.** Проставь 🔴/🟡/🟢 по рубрике конвенций (🔴 — детерминированный слом id без
   product-side алиаса; 🟡 — generic контейнер+роль / i18n-текст / элемент за новым состоянием; 🟢 —
   флоу-дрейф без прямого слома локатора).
6. **Отчёт + (опц.) прогон.** Собери отчёт по шаблону конвенций. Слот доступен → отдай кандидатный
   список спек скиллу `autotest_run` **только на прогон** (не на починку); pass/fail сверни в отчёт
   (🟡→🔴 или снятие ложных). Нет слота → пометь `[не подтверждено прогоном: нет слота]`.

## Выход
- Файл `autotest-impact.md` в папке задачи (`tasks/<слаг>/`); standalone без папки —
  `autotest-impact-<branch>.md` в рабочей папке.
- Вердикт-строка + находки по шаблону: спека/файл · **почему** (корневая причина) · уверенность ·
  **рекомендация QA** · **product-side альтернатива** (если есть).
- **Хэндофф QA:** подготовь файл + короткую сопроводиловку (что за изменение, сколько спек, суть рисков);
  человек копирует и постит в ⟪ADAPT: чат-канал⟫ (как ретро/QA-хэндофф оркестратора). **Агент сам никуда
  не постит** и тесты не редактирует.

## Правила
- **Никогда не редактировать файлы e2e-сьюты** — только отчёт. Это суть скилла.
- **Product-side аккомодации** (сохранить/алиаснуть `data-test`, сохранить лейбл) — **предлагать** как
  более дешёвую альтернативу правке теста, но не делать молча; проходят обычный green-sequence.
- `autotest_run` — звать **только на прогон-подтверждение**, не на починку.
- **Не выдумывать** id/матчи — только фактические из диффа и грепа сьюты.
- Реальный ассерт-фейл при прогоне (не локатор) → возможно, баг фронта: **не подгонять**, зафиксируй
  находку (дальше — Трек B), как учит `autotest_run`.
- Не заменяет полный QA-разбор MR (`qa_mr`) или стейдж-смоук (`delivery_qa_smoke`) — это точечный чек на
  стыке «дифф ↔ сьюта».
- Язык ответа = язык пользователя (по умолчанию русский); английский — только в том, что уходит в репо.
